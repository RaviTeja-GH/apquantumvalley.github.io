Data & Analytics

Understanding information to make better decisions

What is Data?

Data is raw information collected from users, systems, applications or devices. Examples include website visits, login records, transaction details and application logs.

What is Data Analytics?

Data Analytics is the process of examining and analyzing data to discover patterns, trends and useful insights that help organizations make decisions.

Data Types

Structured Data

Structured data is organized in rows and columns and stored inside databases. It follows a fixed schema and can be easily searched using SQL queries. Examples: customer tables, login records, bank transactions, employee databases.

Semi-Structured Data

Semi-structured data does not follow strict tables but still has some organization using tags or keys. It is commonly used in APIs and web applications. Examples: JSON files, XML responses, API responses, configuration files.

Unstructured Data

Unstructured data has no predefined format and cannot be stored in traditional database tables. It usually requires processing or analysis tools. Examples: images, videos, audio files, emails, chat messages, social media posts.

Tools Used

Data Pipeline (VERY IMPORTANT for Support Roles)

What is a Data Pipeline?

A data pipeline is the flow of data from an application to monitoring systems. Every software application continuously generates logs.

Flow:
Application → Logs → Processing → Dashboard → Alerts

Support engineers monitor this pipeline to detect issues before users report them.

Monitoring & Logs

Application Logs

Application logs record normal system activities such as user login, logout, API calls and background processes. Support engineers use these logs to trace user actions and system behavior.

Error Logs

Error logs store failures inside the application such as server errors, database connection failures, timeout errors and exceptions. These logs are critical for debugging production issues.

Monitoring Alerts

Monitoring tools generate alerts when abnormal activity happens. Examples: high CPU usage, server down, API failures or slow response time. Support teams receive alerts and investigate immediately.

Why Data Analytics is Important
